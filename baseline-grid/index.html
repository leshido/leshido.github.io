Baseline Grid

Simple CSS+SVG solution to help with vertical rhythm

As web designers, we are constantly concerned with the concept of rhythm. Rhythm gives clarity and order to our designs, making our websites more readable and less cluttered.

I tend to think of rhythm as two separate, yet interlaced, spacing systems in both axes. On the horizontal axis, many designers use some form of a grid (i.e the famous 960 grid system). This underlying structure helps break the content to multiple pieces, each carefully spaced from the others by a uniform gutter.

The second axis is, of course, the vertical one. Here, rhythm is more easily ignored. While laying out the content horizontally is, usually, a manner of floating divs (or, more recently, flexbox), properly aligning content on a vertical axis is, mainly, a careful consideration of web typography. Because most content on the web is text, evident by the fact we have so many text related HTML tags, vertical rhythm is mostly formed by the relation of these tags to one another.

A great article by Richard Rutter, ‘Compose to a Vertical Rhythm’, goes into far more detail on how to implement a visually pleasing vertical rhythm in web design. However, one caveat is that vertical rhythm is hard to measure. In Adobe InDesign, and other page layout tools, we usually have a simple command to show the baseline grid. In web design, though, we don’t have it that easy.

A nice tool I found to tackle this issue for me is called Basehold.it. It does a very simple thing - it displays a grid of vertical lines over the entire page. This is great for aligning things and making sure that the vertical rhythm is maintained throughout. However, I soon had some minor troubles with the service, mainly related to their servers taking too long to respond. I realized there must be some way to host the service locally and quickly decided to replace their image generation code with a far simpler solution: SVG.
